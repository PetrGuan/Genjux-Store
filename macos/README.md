# Genjux-Store macOS App

Native AppKit app (Phase 1, see [`../ROADMAP.md`](../ROADMAP.md) and [`../.copilot-workflow/PLAN.md`](../.copilot-workflow/PLAN.md) section 6.1), programmatic UI only — no Storyboards/XIBs, no SwiftUI. A thin client of the core service's local HTTP API (see [`../core`](../core)); it does not talk to GitHub directly itself.

## Building

The Xcode project itself is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from [`project.yml`](project.yml) and is **not** committed — regenerate it after pulling or after editing `project.yml` / adding or removing source files:

```bash
brew install xcodegen  # if you don't already have it
cd macos
xcodegen generate
open GenjuxStore.xcodeproj
```

Or build from the command line:

```bash
cd macos
xcodegen generate
xcodebuild -project GenjuxStore.xcodeproj -scheme GenjuxStore -configuration Debug build
```

The app expects a `genjuxd` core service reachable via the local HTTP API (see [`../core/src/lifecycle.rs`](../core/src/lifecycle.rs) for the lazy-start/discovery mechanism the CLI already uses — the GUI's equivalent client lands in a later issue).
