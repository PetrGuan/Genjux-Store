# Request

新项目 Genjux-Store（**开源**，MIT License；最初提议为闭源，后经讨论确认改为开源以建立"下载并执行代码"这类工具所需的信任，并借助社区力量分担 4 平台 UI 的工程量）：一个跨平台（macOS / Windows / Linux / Android）软件商店类客户端，解决"开源网站（GitHub / Gitee 等）不方便安装开源软件"的痛点。

核心形态：
- 根据当前运行平台，自动展示 recommended 软件（例如在 macOS 上运行时展示推荐的 macOS 软件）。
- 用户可以一键安装/试用某个开源项目。
- 安装机制：直接代理/下载官方 GitHub Release 的原始产物，在本地执行安装（不重新打包/不托管转发二进制），以降低法律与安全责任。

架构要求：核心组件（发现、元数据解析、下载、校验、安装编排等业务逻辑）做成跨平台共享库，不同平台只需实现平台特定的 UI 层和平台安装调用。

当前阶段：四个平台同时规划整体架构，暂不写代码。产出应覆盖：
1. 整体分层架构与跨平台核心技术选型（对比方案并给出推荐）
2. 各平台 UI 层技术选型（macOS/Windows/Linux/Android）与和核心的绑定方式
3. Release 产物识别与元数据方案（如何判断某个 asset 对应哪个平台/如何标准化，如缺失元数据时的兜底策略）
4. 安装编排设计：下载、校验（签名/checksum）、沙箱试跑、各平台原生安装调用方式的差异
5. 安全与信任模型（防止分发/触发恶意二进制的风险控制）、以及需要用户确认的边界
6. MVP 范围建议与分阶段路线图（不假设从哪个平台开始，需给出评估与建议）
7. 已知竞品（Homebrew/Winget/Scoop/Snap-Flatpak/F-Droid 等）差异化定位
